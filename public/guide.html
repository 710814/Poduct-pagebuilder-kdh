PageGenie 가이드

초기 설정 사용 순서 저장 방식 템플릿 가이드

AI 기반 상세페이지 자동 생성기
PageGenie에 오신 것을 환영합니다!

PageGenie는 상품 이미지 한 장과 간단한 정보만으로 매력적인 상세페이지를 기획하고, AI(Gemini)를 통해 연출 컷을 자동으로 생성해주는 강력한 툴입니다. 처음 오신 분들을 위해 차근차근 사용법을 안내해 드립니다.

시작하기

1. 시작하기 전에 (필수 초기 설정)

생성된 이미지를 안전하게 구글 드라이브에 저장하고, 데이터를 시트로 관리하려면 가장 먼저 연동 설정이 필요합니다.

설정 메뉴 들어가기

앱 화면의 요소 중 우측 톱니바퀴 아이콘() 버튼을 클릭하여 설정 창을 엽니다. 설정이 비어있을 경우 톱니바퀴에 빨간 점이 표시됩니다.

연동 정보 입력하기

  • Google Sheet ID: 연동하고자 하는 구글 시트 URL의 중간에 있는 긴 문자열.
  • GAS URL: 배포된 구글 앱스 스크립트(Google Apps Script)의 웹 앱 URL.

이 두 가지를 정확히 입력하고 '저장'을 누르면 클라우드 저장 준비가 끝납니다.

2. 상세페이지 제작 4단계 가이드

STEP 1

모드 선택

원하시는 작업 방식(모드)을 선택합니다. AI를 활용하여 무에서 유를 창조하는 '생성 모드'를 선택하여 다음 단계로 넘어갑니다.

STEP 2

정보 및 원본 입력

상품의 메인 이미지를 업로드하고, 상품명, 브랜드, 색상 옵션, 사이즈 등의 상세 스펙과 대중에게 어필할 소구점 3가지를 기입합니다. 작성한 내용은 AI가 이미지를 그릴 때 중요한 힌트가 됩니다.

STEP 3

템플릿 & 이미지 생성

제공되는 템플릿(레이아웃)을 선택하면 앱이 각 구역에 들어갈 이미지 프롬프트(명령어)를 자동 설계합니다. 내용을 확인한 후 "이미지 생성" 버튼을 누르면 AI가 그림을 그리기 시작합니다.

STEP 4

결과 확인 및 완료

생성 완료된 이미지를 미리보기로 확인합니다. 마음에 든다면 DB/이미지 저장 버튼이나 HTML 다운로드 버튼을 눌러 작업을 최종 마무리합니다.

3. 이미지 저장 최적화 (가장 중요)

💡 언제 이미지가 구글 서버에 저장되나요?

3단계에서 단순히 "이미지 생성"을 연속으로 누른다고 해서 그 즉시 구글 시트나 드라이브에 저장되어 요금이나 용량을 낭비하지 않습니다. 생성된 이미지는 임시로 사용자의 웹 브라우저 메모리에만 보관됩니다. (미리보기 용도)

오직 모든 생성결과가 마음에 들어 4단계(결과 화면)에 진입한 후, DB/이미지 저장 버튼을 사용자가 직접 누를 때에만, 단 1회에 걸쳐 일괄적으로 서버에 업로드 및 기록됩니다.

옵션 A: DB/이미지 저장 버튼

  • 구글 드라이브에 상품명 폴더 자동 체계화
  • 모든 이미지를 해당 폴더로 일괄 업로드
  • 구글 시트의 새 행(Row)에 프롬프트 텍스트 이력 및 이미지 다운로드 링크 기록
  • * 팀 협업이나 히스토리 보관 목적으로 가장 강력히 권장됩니다.

옵션 B: HTML 다운로드 버튼

  • 즉시 내 컴퓨터로 압축파일(ZIP) 다운로드
  • 인터넷이나 구글 연동 없이 로컬 이미지와 HTML을 보유
  • 쇼핑몰 관리자 페이지에 그대로 복사붙여넣기 할 때 사용
  • * 구글 시트 기록이 남지 않으며, 완전한 로컬 단독 저장이 필요할 때 사용합니다.

4. 템플릿(Layout)의 이해

템플릿은 상세페이지의 '뼈대'입니다. 섹션 단위로 나누어져 있으며, 각 템플릿의 속성에 따라 생성되는 이미지의 갯수와 배열이 달라집니다.

단일 이미지 (Single)

세로로 길거나 가로로 꽉 찬 하나의 큰 이미지를 생성합니다. 브랜드 인트로보드나 큰 배너 영역에 적합합니다.

그리드 (Grid)

2칸, 3칸, 4칸 등 동일한 비율의 이미지를 격자 형태로 나열합니다. 컬러칩이나 디테일 컷을 여러 장 보여줄 때 매우 유용합니다.

콜라주 (Collage)

크고 작은 이미지가 불규칙적으로, 그러나 감각적으로 섞여있는 레이아웃입니다. 라이프스타일 컷과 룩북 분위기를 낼 때 사용합니다.

기본 템플릿 지정하기

매번 템플릿을 선택하기 귀찮으신가요? 템플릿 선택 창에 보시면 [기본값으로 설정] 버튼이 있습니다. 자주 쓰시는 레이아웃을 등록해두면 다음부터는 자동으로 해당 템플릿이 최상단에 준비됩니다.

나만의 템플릿 만들기 (고급 설정)

제공되는 기본 템플릿 외에도, 사용자가 직접 원하는 구성으로 템플릿을 만들거나 수정할 수 있습니다. 템플릿 편집은 앱 우측 상단 설정() 메뉴의 [템플릿 설정] 탭에서 가능합니다.

1

새 템플릿 추가하기

템플릿 목록 화면에서 언제든 새 템플릿을 추가할 수 있습니다.

  • 이미지 분석으로 추가 (AI 자동 생성):

    참고하고 싶은 타사 상세페이지나 잘 만들어진 이미지를 업로드하세요. AI가 이미지의 레이아웃 구조를 분석하여 템플릿(섹션 갯수, 그리드 등)을 자동으로 생성해 줍니다.

  • 직접 구성 (빈 템플릿 추가):

    "직접 구성" 버튼이나 원하시는 카테고리(뷰티, 패션 등)를 선택하여 처음부터 구조를 짜 올릴 수 있는 빈 템플릿을 엽니다.

2

섹션(Section) 구성 및 관리

하나의 긴 상세페이지는 여러 개의 '섹션' 블록이 아래로 쌓이면서 완성됩니다. 편집 화면 중앙과 우측에서 섹션을 관리합니다.

  1. 섹션 추가: 템플릿 아래쪽의 + 섹션 추가 버튼을 눌러 새 영역을 만듭니다. 가장 최상위에는 상품의 메인 타이틀을 담당하는 'Top Title'을 하나 추가하는 것을 권장합니다.
  2. 섹션 이름: 우측 설정 패널의 "섹션 이름" 입력창에서 해당 영역의 역할(예: 메인 배너, 디테일 컷, 소재 설명)을 알아보기 쉽게 적습니다. 이 이름은 결과 화면에 제목으로 표시됩니다.
  3. 설명/부제목: 섹션의 부제목이나 간략한 설명(Description)을 입력하면, 결과 이미지와 함께 텍스트로 보일 수 있습니다. (테마에 따라 표시 방식이 다름)
  4. 레이아웃: 이 섹션의 모양을 결정합니다. "Single(단일 이미지)", "Grid(바둑판 배열)", "Collage(크고 작은 이미지 혼합)" 중 하나를 선택합니다.
  5. 섹션 순서 변경 및 삭제: 중앙 패널의 섹션 블록 우측 상단에 있는 화살표 아이콘()으로 위아래 순서를 바꾸거나, 휴지통 아이콘()으로 삭제할 수 있습니다.
3

섹션 내 이미지 슬롯(Slot) 설정

섹션의 레이아웃을 정했다면, 그 안에 들어갈 각각의 이미지 영역(슬롯)을 세부적으로 지시해야 합니다. 우측 패널의 '섹션 내부 이미지 구성' 영역에서 설정합니다.

자동 프롬프트 vs 고정 프롬프트
  • 기본 슬롯 (연필 아이콘 ): 모델이 앱 프론트에서 소구점을 기반으로 알아서 동적 영문 프롬프트를 작성하게 둡니다. (가장 많이 씀) 슬롯 이름만 '디테일 컷', '활용 모습' 등으로 적어주세요.
  • 고정 프롬프트 (자물쇠 아이콘 ): 이 자리에 들어갈 이미지는 AI가 마음대로 바꾸면 안 될 때 사용합니다. 자물쇠 아이콘을 눌러 고정 모드로 바꾼 뒤, "A close-up shot of..." 처럼 직접 확정된 영문 프롬프트를 적어넣어 고정시킵니다.
비율 (Aspect Ratio) 선택

사진 한 장 한 장의 가로세로 비율을 결정합니다. (예: 1:1, 4:5, 16:9)

주의: Grid나 Collage 레이아웃은 선택하신 비율의 크기에 맞춰 화면 분할 코드를 생성하므로 가급적 일관된 비율을 설정하는 것이 깔끔하게 나옵니다.

슬롯 추가/삭제하기

"슬롯 추가(+)" 버튼을 눌러 이 섹션에 포함할 이미지 수를 늘리거나 뺄 수 있습니다. 원하는 이미지 만큼만 딱 맞춰 구성하세요.

모든 수정이 끝났다면 가장 우측 상단의 수정 완료 버튼을 눌러 저장하는 것을 잊지 마세요!

자주 묻는 질문

Q. 수십 장을 한번에 만들다 보니 완료가 너무 오래 걸려요.

A. 구글 최신 AI 모델을 사용하므로 다량의 고화질 이미지 렌더링 시 대기 시간이 발생합니다. 렌더링 도중 창을 닫거나 새로고침하지 마시고 잠시 대기해주세요. 진행바(Progress bar)를 통해 상황을 모니터링할 수 있습니다.

Q. DB 저장 중 "전송 용량이 너무 큼" (텍스트만 저장) 오류가 떴어요.

A. 구글 서버(GAS)는 한번에 보낼 수 있는 최대 데이터 한계(약 50MB)가 존재합니다. 고해상도 이미지가 너무 많을 경우 발생하는 안전 장치입니다. 이 경우 해당 DB 기록에는 이미지 파일이 누락되었을 수 있으므로 [HTML 다운로드]를 통해 로컬에 원본을 보관하는 것을 권장합니다.

Q. 생성된 모델/상품 이미지의 색상이 약간 다르게 나와요.

A. AI는 텍스트(프롬프트)와 제공된 원본 이미지를 혼합하여 연출컷을 재창조합니다. 2단계 정보 입력란에 색상 옵션을 명확히 기재해주시고, 필요시 각 컬러별 "후면/앞면/기타" 이미지를 정확히 매칭해 업로드 하시면 정확도를 훨씬 더 높일 수 있습니다.
