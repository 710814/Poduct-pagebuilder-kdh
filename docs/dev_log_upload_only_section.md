# 개발 내역 및 버그 수정: 업로드 전용 섹션 기능 구현

## 1. 개요
*   **일시:** 2026-03-18
*   **목적:** 템플릿 설정에서 사용자가 이미지 슬롯을 모두 삭제한 섹션에 대해, AI 프롬프트 생성 및 이미지 생성을 방지하고 사용자 업로드 전용으로 동작하게 함.

## 2. 주요 작업 내역
1.  **AI 분석 단계 (`geminiService.ts`) 수정**
    *   `generateImageSlotsForLayout` 함수: `existingSlots`가 빈 배열(`[]`)인 경우 명시적인 슬롯 삭제로 간주하고 새로운 슬롯을 자동 생성하지 않도록 변경.
    *   `applyTemplateStructure` 함수: 이미지 슬롯이 없고 프롬프트도 없는 섹션을 "업로드 전용"으로 식별하여 `baseImagePrompt`를 비워두도록 처리.
2.  **화면 표시 (`StepAnalysis.tsx`) UI 개선**
    *   "업로드 전용" 섹션(`isUploadOnlySection`) 판별 로직 추가.
    *   해당 섹션에서는 AI 프롬프트 입력창, 스타일 설정, 재생성 버튼을 숨김.
    *   빈 이미지 슬롯일 때 이미지 업로드를 유도하는 전용 UI(안내 문구 및 카메라 아이콘 클릭 영역 확장 등) 제공.
    *   이미 업로드된 이미지가 있는 경우 불필요한 액션 버튼 제거(크게 보기, 삭제만 남김).
3.  **최종 생성 단계 (`App.tsx`) 로직 변경**
    *   `handleGenerate` 함수: 실제 이미지 생성 대상(`sectionsToGenerate`)을 필터링할 때 `isUploadOnlySection` 조건 추가하여 생성 대기열에서 제외시킴.
    *   AI 처리 완료 후 생성된 섹션을 모을 때 원본 섹션 상태 그대로 `newSections` 배열에 추가하여 사용자 업로드 결과물이 덮어씌워지지 않게 함.

## 3. 해결된 버그 및 사용자 피드백
*   **이슈:** 이미지 슬롯을 0개로 설정해도 레이아웃의 종류에 따라 AI가 1개 이상의 슬롯을 임의로 복구해버리거나 프롬프트를 자동 생성해버리는 문제 발생.
*   **해결:** 코드 상에서 의도적인 빈 배열과 아직 설정되지 않은 상태를 명확히 구분함으로써 사용자 의도를 100% 반영하도록 수정 완료.
